## Project Overview

This is an ESP-IDF 5.5 project driving an SSD1322 256x64 grayscale OLED display using LVGL v9.4.0 for UI rendering. The UI is generated by EEZ Studio.

## Build Commands

```bash
# Set target chip (e.g., esp32c3, esp32s3)
idf.py set-target <chip>

# Configure project (SPI pins, LVGL settings via menuconfig)
idf.py menuconfig

# Build
idf.py build

# Flash to device
idf.py -p /dev/ttyUSB0 flash

# Monitor serial output
idf.py -p /dev/ttyUSB0 monitor
```

## Architecture

The initialization flow in `main.c` establishes three layers:

1. **SSD1322 Driver** (`ssd1322_driver.c/h`) - Low-level SPI communication with the OLED. Handles GPIO (DC, RST) and SPI bus configuration. Provides `ssd1322_send_cmd()` and `ssd1322_send_data()` primitives.

2. **LVGL Adapter** (`lvgl_adapter.c/h`) - Bridge between LVGL and the SSD1322. Creates an LVGL display with L8 color format (8-bit grayscale), allocates buffers, and implements `lvgl_flush_cb()` which converts L8 pixels to I4 format (4-bit grayscale, 2 pixels per byte) before sending to the display via SPI polling.

3. **UI Wrapper** (`ui_wrapper.c/h`) - Initializes the EEZ Studio-generated UI and signals LVGL when ready. The `ui/` directory contains all EEZ Studio output: `screens.c/h` (screen definitions), `styles.c` (LVGL styles), `images.c` (image assets), `ui.c` (UI initialization/tick functions).

### Key Files
- `main/ssd1322_driver.h` - GPIO/SPI pin definitions (PIN_NUM_MOSI, PIN_NUM_CLK, etc.)
- `main/lvgl_adapter.c:18` - `lvgl_flush_cb()` - L8 to I4 color conversion logic
- `main/ui_wrapper.c:7` - `ui_wrapper_init()` calls EEZ's `ui_init()` then signals LVGL

### Color Format Note
SSD1322 uses I4 format (4 bits per pixel, 2 pixels per byte). LVGL renders in L8 format (8 bits per pixel). The adapter at `lvgl_flush_cb()` converts by taking high 4 bits of consecutive L8 pixels and packing them into I4 bytes.

## Hardware Configuration

Default GPIO mappings (defined in `ssd1322_driver.h`):
- MOSI: IO12
- CLK: IO3
- CS: IO0
- DC: IO1
- RST: IO10
- SPI Clock: 10MHz
