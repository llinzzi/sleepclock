# Sleep Clock PRD

## 1. Concept & Vision

一款床头氛围睡眠时钟。平日沉默如夜，按键唤醒时，柔和的灰阶渐显时间，仿佛月光从屏幕中渗出。不仅是时钟，更是一件能与环境对话的夜间艺术品。

结合双路氛围灯、舵机结构、全频扬声器，这是一台能"动"、能"响"、能"发光"的多感官唤醒设备。

## 2. Design Language

### 视觉风格
- **暗黑美学**: 平时屏幕完全熄灭，仅有微弱的状态指示灯
- **月相灵感**: 唤醒时使用深灰到浅灰的渐变，模拟月光质感
- **极简主义**: 只显示必要信息——时间，无多余元素

### 颜色方案（I4 灰阶）
- **纯黑** `0x0` - 背景，完全熄灭
- **深灰** `0x5` - 次要信息
- **中灰** `0x8` - 时间数字阴影
- **浅灰** `0xB` - 时间数字主体
- **亮灰** `0xF` - 高亮/强调

### 字体
- 时间显示: 粗体大号数字，浅灰色
- 状态指示: 小号点阵字符，深灰色

### 动效哲学
- **缓入**: 按键触发后 800ms 渐显，避免刺眼
- **呼吸**: 显示期间时间数字有微弱亮度呼吸感 (±5% 亮度周期 4s)
- **缓出**: 无操作 10 秒后 1200ms 渐隐回全黑

## 3. Layout & Structure

## 6. 硬件利用思考

### 6.1 SSD1322 灰阶的创意用法

| 效果 | 实现方式 | 视觉描述 |
|------|----------|----------|
| 月光渗出 | 8级灰度渐显 | 时间从黑暗中缓缓浮现，如月光穿透云层 |
| 深度感 | 数字使用2-3层灰度叠加 | 时间数字有轻微3D浮雕感 |
| 氛围底噪 | 全屏极淡噪点纹理 | 避免纯黑死屏，增加有机感 |
| 呼吸光晕 | 整屏亮度±5%正弦波动 | 模拟呼吸节奏的微弱光效 |
| 星辰散落 | 随机淡灰点点缀背景 | 唤醒时短暂显示，如繁星闪烁 |

### 6.2 日间天气可视化

屏幕纯黑背景，显示未来12小时逐时预报：

```
┌─────────────────────────────────┐
│  25°  ●━━●━━●━━●━━●━━●━━●    │  ← 温度曲线 + 原点
│                                 │
│  8:00  9  10  11  12  13  14 15 │  ← 时间轴（首个显示分:秒）
└─────────────────────────────────┘
```

**温度曲线**: 从左到右显示12小时的温度变化趋势，点在线上。

**雨滴动画**: 有降水概率（≥30%）的小时，显示雨滴从屏幕上方下落：
- 雨滴速度 = 10 + (当前降水概率 / 5)
- 雨滴数量 = 降水概率 / 10
- 雨滴间距 = max(8, 20 - 降水概率 / 10)

**时间轴**: 首个时间点（当前时间）显示"时:分"格式，其余只显示"时"。

### 6.4 双路氛围灯 (GPIO20/21 → FM116C)

两路灯光可以独立控制，实现丰富的光效：

| 场景 | 灯光表现 | 氛围描述 |
|------|----------|----------|
| **日出唤醒** | 暖光(路1)从0%→100%在30分钟内逐渐亮起 | 模拟真实日出，温和唤醒 |
| **日落助眠** | 冷光(路2)先亮，暖光(路1)逐渐替代 | 模拟日落，促进褪黑素分泌 |
| **星辰投影** | 两路交替闪烁，模拟星光 | 将天花板变成星空 |
| **呼吸同步** | 两路灯光随呼吸节奏明暗交替 | 引导用户呼吸放松 |
| **环境跟随** | 灯光随OLED显示亮度联动 | 屏幕亮则灯亮，屏幕暗则灯暗 |
| **双色温阅读** | 冷暖两路混合调节色温 | 床头阅读灯模式 |
| **微光守护** | 一路保持微弱常亮(5%) | 夜间指引，不完全熄灯 |

> **功耗注意**: FM116C 是 H 桥驱动，适合驱动 LED 灯带或小功率灯泡，每路建议不超过 500mA。

### 6.5 扬声器 (GPIO4/5/6 → NS4168 I2S)

I2S 音频功放可以输出丰富的声音：

| 场景 | 声音表现 | 体验描述 |
|------|----------|----------|
| **自然唤醒** | 鸟鸣、溪流、风声渐起 | 比刺耳闹铃更符合生理节律 |
| **渐进闹铃** | 音量从 0% 在 5 分钟内渐升至 80% | 温和唤醒，不惊吓 |
| **白噪音助眠** | 雨声、海浪声、粉红噪音 | 屏蔽环境噪音帮助入睡 |
| **整点报时** | 每小时一次轻柔报时音 | 不看手机也知道时间 |
| **呼吸引导** | 特定频率提示音配合灯光 | 引导冥想/呼吸练习 |
| **闹铃音乐** | 内置几首轻音乐作为闹钟 | 可爱的复古闹铃声 |

> **省电**: 深度睡眠时关闭功放电源 (GPIO2 NS_CTRL)。

### 6.6 舵机结构 (GPIO9)

GPIO9 复用为舵机控制，可以做以下物理交互：

| 玩法 | 结构描述 | 体验描述 |
|------|----------|----------|
| **日出窗帘** | 舵机拉绳逐渐拉开窗帘 | 物理方式让阳光唤醒，比灯光更真实 |
| **纸鹤风车** | 舵机带动小型纸鹤/风车旋转 | 轻柔的视觉提醒，如梦境飘动 |
| **反光镜云朵** | 舵机旋转带动天花板反光点移动 | 星光投影效果，照亮屋顶 |
| **指针时钟** | 舵机带动时针/分针旋转 | 物理与数字结合的复古美感 |
| **轻敲提醒** | 舵机带动小锤轻敲桌面 | 无声的触觉提醒 |
| **翻页显示** | 舵机翻动小型翻页结构 | 每分钟翻一页，极简主义 |

> **注意**: GPIO9 是 ESP32-C3 的 Boot Strapping 引脚，上电时需保持正确电平。运行时作为舵机 PWM 控制没问题。

### 6.7 按键 (GPIO3)

| 交互 | 动作 |
|------|------|
| 单击 | 唤醒屏幕 + 氛围灯 |
| 双击 | 快速切换氛围灯场景 |
| 长按 3秒 | 进入设置模式 |
| 长按 10秒 | 强制重启 |

## 7. 场景组合

### 唤醒场景示例

**周一清晨 7:00 唤醒：**
1. **06:30** - 暖光开始 30 分钟缓慢渐亮（日出模拟）
2. **07:00** - 舵机轻拉窗帘，物理唤醒
3. **07:00** - 闹铃渐响，自然声音
4. **07:05** - OLED 屏幕渐显时间
5. **07:10** - 用户按键确认起床，灯光保留

**夜间入睡：**
1. 用户单击按键
2. 氛围灯切换暖光（路1:100%, 路2:20%）
3. OLED 显示时间 + 星辰背景动效
4. 白噪音/雨声播放
5. 10 秒无操作，灯光 + 屏幕渐隐
6. 30 分钟后进入深度睡眠

### 9.3 模块要求

#### 9.3.1 显示模块 (ScreenMgr)

**文件**: `main/ui/screens.c` 或新建 `screen_mgr.c/h`

**显示模式枚举**:
```c
typedef enum {
    SCREEN_SLEEP,      // 待机（黑屏）
    SCREEN_WAKE,       // 唤醒（自动根据时段选择）
    SCREEN_STAR,       // 星辰模式（20:00-06:59）
    SCREEN_WEATHER,    // 天气模式（07:00-19:59）
} screen_mode_t;
```

**亮度管理**:
- `brightness`: 0-15 整数，对应 I4 灰阶
- 渐变步进: 每次 +0.3 或 -0.5
- 呼吸效果: `sin(breathPhase * 0.5) * 0.5 + 0.5`，周期 4s

**星辰模式渲染**:
1. 随机生成 30 个星点，位置 0-255(x) × 0-63(y)
2. 星点大小: 70% 为 1×1，30% 为 2×2
3. 星点亮度: 2-5 级灰度
4. 闪烁效果: `sin(breathPhase * 3 + star.x) * 0.5 + 0.5`
5. 时间显示: 28px 粗体 Orbitron 字体，垂直居中(y=32)

**星辰模式动画时序**:
| 时间 | 星星 | 时间 |
|------|------|------|
| 0-3s | 淡入 (0→12) | - |
| 2-4s | 淡入中 | 淡入 (0→15) |
| 4-10s | 全亮 | 全亮 |
| 10-12s | 全亮 | 淡出 (15→0) |
| 12-13s | 全亮 | 消失 |
| 13-15s | 淡出 (12→0) | 消失 |
| 15s+ | 消失 | 消失 |

**天气模式渲染**:
1. 背景: 纯黑
2. 温度曲线: 从左到右 12 个数据点
   - X 位置: `14 + i * 20`（i = 0-11）
   - Y 位置: 根据温度在 8-45 范围内映射
3. 温度标签: 始终显示在点上方
   - 位置: y - 6，最小 y=10 避免顶出界
4. 时间轴: y=48 横线
5. 小时标签: y=56
   - 首个（当前时间）: "H:MM" 格式，粗体
   - 其余: "H" 格式
6. 雨滴动画: 当 `rain_prob >= 30`
   - 速度: `10 + rain_prob / 5`
   - 数量: `rain_prob / 10`
   - 间距: `max(8, 20 - rain_prob / 10)`

```
│    │ - 绘制时间轴和小时标签            │ │
│    └──────────────────────────────────┘ │
├─────────────────────────────────────────┤
│ 3. breathPhase += 0.02 (呼吸动画)         │
│ 4. lv_timer_handler()                   │
└─────────────────────────────────────────┘
```

### 9.5 星辰模式渲染详细规格

| 元素 | 规格 |
|------|------|
| 星点数量 | 30 个 |
| X 范围 | 0-255 随机 |
| Y 范围 | 0-63 随机 |
| 大小分布 | 70% 为 1×1，30% 为 2×2 |
| 亮度范围 | 2-5 级灰度 |
| 闪烁效果 | `sin(breathPhase * 3 + star.x) * 0.5 + 0.5` |
| 时间字体 | Bold 28px Orbitron |
| 时间位置 | 垂直居中 x=128, y=32 |
| 阴影偏移 | +1,+1 像素，亮度-3 |
| 状态点 | y=50, 半径1.5, 亮度5-7呼吸 |

### 9.6 天气模式渲染详细规格

| 元素 | 规格 |
|------|------|
| 数据范围 | 未来12小时 (hourlyForecast[0-11]) |
| X 位置 | `14 + i * 20` (i=0-11) |
| Y 范围 | 8-45 (根据温度映射) |
| 温度曲线 | 1px 连线，亮度=textBrightness |
| 数据点 | 圆点，半径2，亮度+2 |
| 温度标签 | 7px Bold，显示在点上方或下方 |
| 时间轴 | y=48, 1px 横线 |
| 小时标签 | 6px，y=56 |
| 首个时间 | "H:MM" 粗体格式 |
| 其余时间 | "H" 格式 |
| 雨滴条件 | rain_prob >= 30 |
| 雨滴速度 | `10 + rain_prob / 5` |
| 雨滴数量 | `rain_prob / 10` |
| 雨滴间距 | `max(8, 20 - rain_prob / 10)` |
| 雨滴大小 | 1×4 像素 |

### 9.7 WiFi 和 NTP 要求

**连接策略**:
1. 上电后尝试连接已知 WiFi
2. 失败则进入配网模式 (SmartConfig/AP)
3. 连接成功后立即 NTP 校时
4. 每小时 NTP 校时一次

**NTP 服务器**:
- 首选: `ntp.aliyun.com`
- 备用: `ntp.tencent.com`
- 时区: +8 小时

### 9.9 关键实现检查点

| 检查点 | 验证方法 |
